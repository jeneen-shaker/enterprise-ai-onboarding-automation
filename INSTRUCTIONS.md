# Submission Instructions

## What to submit

Please include the following in your fork:

### For Task 1
Add your written solution as:

`starter/design-solution.md`

Your document should cover:
- Step-by-step workflow logic
- AI usage across the onboarding flow
- Prompt engineering details
- Integrations and data flow
- Operational benefits and expected impact

### For Task 2
Add your prototype or scaffold materials in one or more of these folders:
- `starter/workflows/`
- `starter/code/`
- `starter/diagrams/`
- `starter/screenshots/`

Examples:
- n8n workflow export JSON
- Zapier screenshots
- Python or JavaScript helper scripts
- Architecture or workflow diagrams
- Loom walkthrough link inside your submission template

### Submission template
Complete:
`SUBMISSION_TEMPLATE.md`

## Pull Request format

Please title your PR as:

`Submission - [Your Full Name]`

In the PR description, include:
- Your full name
- Your email address
- Short summary of your solution
- Any assumptions made
- Any setup instructions

## Notes
- Keep your submission clear and well organized
- Practical solutions are preferred over overly complex ones
- You may use mock data and placeholder integrations
- You may use any LLM, automation platform, or code stack you prefer
